# Project Brief

## Goal

Build a consumer family travel site that helps parents plan trips with kids using destination-specific SEO pages and interactive AI planning tools.

## Audience

Parents and caregivers planning trips with babies, toddlers, kids, tweens, or teens. The user often has practical constraints that generic travel guides ignore:

- Nap schedules
- Stroller access
- Car seats
- Transit complexity
- Hotel room size
- Pool and breakfast needs
- Rainy-day backups
- Budget
- Walking tolerance
- Multi-child age gaps

## Core Promise

Help families quickly answer:

- Where should we stay?
- What should we do with kids this age?
- Which hotel is actually family-friendly?
- Can we do this itinerary without exhausting everyone?
- What should we avoid?
- What is the backup plan if weather, naps, or transit go sideways?

## SEO Wedge

The strongest initial opportunity is destination-specific family travel queries:

- `things to do in {destination} with kids`
- `where to stay in {destination} with kids`
- `best family hotels in {destination}`
- `{destination} itinerary with kids`
- `{destination} with toddlers`
- `{destination} with teens`

## Product Wedge

Compete with static blogs by making pages interactive:

- Area recommender
- Hotel review summarizer
- Nap-friendly itinerary builder
- Stroller/transit difficulty scoring
- Rainy-day planner
- Map export
- Packing list generator
